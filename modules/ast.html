resolveByName

  • resolveByName<T>(scope: ContractDefinition, constructor: ASTNodeConstructor<T>, name: string, onlyParents?: boolean): T[]
    • Defined in ast/dispatch.ts:86

    Resolve an inheritable contract property following the standard C3 linearization order.

    Type parameters

    • T: Resolvable

    Parameters

    • scope: ContractDefinition
    • constructor: ASTNodeConstructor<T>

      The class constructor of the type of contract property we are resolving. One of FunctionDefinition, ModifierDefinition, EventDefinition or VariableDeclaration

    • name: string

      Name of the property we are attempting to resolve. Note that if we are resolving functions or events there may be multiple results as functions/events can have the same name and different arguments

    • onlyParents: boolean = false

      boolean flag specifing that we want to only look through the bases of the contract.

    Returns T[]

split

  • split(string: string, separator: string, openBrace: string, closeBrace: string): string[]
    • Defined in ast/utils.ts:17

    Splits the string using the separator. If the string contains well formed parenthesized expressions then those expression will not be split even if the contain separators.

    E.g. if we call split("a,b,c(d,e)", ",", "(", ")") we will get ["a", "b", "c(d,e)"].

    Parameters

    • string: string

      String to split

    • separator: string

      Separator to use to split the string

    • openBrace: string

      String to consider to be an "opening brace"

    • closeBrace: string

      String to consider to be a "closing brace"

    Returns string[]

    Return the parts into which s has been split.
